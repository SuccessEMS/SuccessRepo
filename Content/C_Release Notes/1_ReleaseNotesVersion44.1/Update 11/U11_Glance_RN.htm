EMS Glance Release Notes
V44.1, Update 11

Important: There is a known iOS issue where homelinks on iPads (i.e. a full screen Safari browser page that is saved as an app) do not always clear their cache even if you do so manually. This issue may cause some of your devices to have errors. If you see an error, please close Glance on the iPad and re-open it using the homelink.

This section describes updates to your EMS software. 

See Also: Contact Customer Support and Updating to the Latest Version of EMS (Patches and Builds).

Name: V44.1 Update 11

Release Date: May 12, 2017

Build #:   17.5.10.13
Glance API: 1.0.1.020

Enhancements

Area

Description

.NET Framework

Enable .NET Framework 4.6.1

Modified functionality so that EMS Glance is compatible with .NET Framework 4.6.1

Event Information

Increase Display Options for Event Information

Modified several settings (in EMS Desktop Client under Glance Settings) to offer more choices for event information that displays on Glance tablets, so users can now see Event Type and Booking Setup Type, depending on parameter settings. Renamed the parameter, "Display Event Name" to "Display Heading 1". Renamed the parameter, "Display Group Name" to "Display Heading 2". Renamed the parameter "Display Contact Name" to "Display Heading 3". Clicking on these parameters now shows a drop-down menu where users can choose from Event Name, Group Name, Contact Name, Event Type, Booking Setup Type, and (none). For Heading 1, the default is Event Name. For Heading 2, the default is Group Name

Permissions

Enable Web Process Template Security

Added a setting (in EMS Desktop Client under Glance Settings) to enable Glance to check User Process Templates when users request space. When set to Yes, this parameter, "Enable Everyday User Process Template Security," requires the user to have access to the room from at least one of his everyday user templates, or the room must be in the unauthenticated user process template. 

New Reservation

Hide End Times That Could Cause Conflicts

Enhanced new reservation screens so that meeting end times no longer display as available when they can cause a booking conflict. 

Fixes 

Area

Description

Check In

Check-In Button Displayed Based On Reservation Time, Not Event Time

The meeting Check-In button became available based on the meeting reservation time when it should have displayed based on the event time.

Fix: Modified functionality so that the Check In button activates based on the set number of minutes before booking start time.
