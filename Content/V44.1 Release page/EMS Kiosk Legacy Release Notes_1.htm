Fixes 

Area

Description

Make a Reservation

Users Could Not Proceed in "Make a Reservation" Page When "Require Timeout to Terminate User Session" Was Set to No

Users in the EMS Kiosk App (Legacy) were unable to make reservations in the Make a Reservation page because they were being sent back to login.aspx if the parameter "Require Timeout/Logoff to Terminate User session'' was set to 'NO.'  This parameter does not affect the authentication step when making reservation, specially when there is no wait time. 

Fix: Modified functionality so that users can proceed to completing their reservation in the EMS Kiosk App (Legacy).

Error Message

If the EMS Kiosk App (Legacy) Was Idle, Users Received An Error Message

Users experienced an unknown error after the EMS Kiosk App (Legacy) had been idle for twice the time Administrators had set for the page timeout. 

Fix: Modified functionality so that users do not receive an error when the EMS Kiosk App (Legacy) is idle.
