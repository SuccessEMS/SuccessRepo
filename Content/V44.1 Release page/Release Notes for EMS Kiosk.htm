Release Notes for EMS Kiosk App, V44.1 Update 18 (EMS Kiosk App)

This section describes updates to your EMS software. 

See Also: Contacting Customer Support and Updating to the Latest Version of EMS

Name: V44.1, Update 18

Release Date: October 31, 2017

Patch: 44.1.18000.201710252

Build #:  44.1.18000.209

Fixes 

Area

Description

Configuration

Entire Page Was Moving When Scrolling on Touch Screen

Users were experiencing the entire page moving when scrolling outside of the profile list on a touch screen device.

Fix: Modified functionality so that entire page does not move.

Inactive Profiles Were Visible in List

Users could still see inactive profiles when searching for them from the Configuration page.

Fix: Modified functionality so that if the "Inactive" checkbox is marked on the required Kiosk profile, it should be hidden in the profiles list on the 'config' page.

Today's Events

Bad Profile Configuration Broke Building and Floor Filters

Building filter was incorrectly defaulting to All Buildings and the floor filter was defaulting to All Floors.

Fix: Modified functionality so that the building filter defaults to the building on the profile and the floor filter defaults to the Default floor.

TFS item 32719

Authorization

"No Results Found" Validation Message Displays by Default on Secondary Auth Screen After Using Invalid Badge on Badge Auth

Users who scanned an invalid Badge and then clicked on the "I don't have my badge" button received a validation message that continued to display by default on the secondary auth screen.

Fix: Modified functionality so that the validation message does not appear on the secondary authorization screen.

TFS item 32782

Locate a Person

The Max Record Returned for Group Searches Parameter Wasn't Being Honored

Users who set the parameter in the EMS Desktop Client were experiencing search results that were not honoring this value.

Fix: Modified functionality so that search results honor the parameter's value.

TFS item 33217

The Max Booking Availability Records Returned Parameter Wasn't Being Honored

Users who set the parameter in the EMS Desktop Client were experiencing search results that were not honoring this value.

Fix: Modified functionality so that search results honor the parameter's value.

TFS item 33247

My Reservation

The Allow User to Cancel/End Booking Outside of Profile Parameter Wasn't Being Honored

Users who set the parameter in the EMS Desktop Client were experiencing search results that were not honoring this value.

Fix: Modified functionality so that search results honor the parameter's value.

TFS item 33267

Profiles

Splash Screen for Successfully Applied Profiles Did Not Automatically Move to Next Screen

Users applying profiles were experiencing a stuck splash screen that would not progress. The splash screen should appear for a few seconds before moving on the next Welcome screen and then on to the default page.

Fix: Modified functionality so that users can progress beyond the splash screen when applying profiles.
