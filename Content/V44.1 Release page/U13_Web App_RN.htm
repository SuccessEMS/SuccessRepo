Patch:  44.1.13000.201707100

Fixes 

Area

Description

Skype for Business

For ADFS Authentication, Popup Window Did Not Close 

Users authenticating with ADFS who successfully logged in were experiencing a popup window that would not close. 

Fix: Modified functionality so that popup window closes after user successfully logs into Skype for Business. 

After Adding a Room to Their Reservation, Users Were Unable to Toggle Skype Icon (EMS Mobile Web App)

In the EMS Mobile Web App, users were unable to toggle the Skype icon after adding a room to their reservation. 

Fix: Modified functionality so that users can toggle the Skype icon after adding a room. 

Room Names with Special Symbols Displayed Incorrectly in Skype Calendar

Skype meetings including room names with special symbols that were created in the EMS Web App were displayed incorrectly in the Skype calendar.

Fix: Modified functionality so that room names with special characters are displayed correctly in the Skype Calendar.

If Adding Skype for Business to a Reservation, Users Cannot Disable Add to Calendar Checkbox

For Skype for Business to function properly, the meeting must be added to the creator's calendar. If the Add to Calendar checkbox is unchecked, Skype for Business would not be added to the reservation. 

Fix: Modified functionality so that  users cannot disable the Add to Calendar checkbox.

Attendees

Booking and Resources Applied Incorrectly

When a reservation was created with multiple locations and an attendee resource type was added to a building the attendee resource was incorrectly added on all locations. 

Fix: Modified functionality so that resources are added correctly. 

Services tab

Unauthenticated Requests Did Not Include a Next Step Button

When a reservation was created with an unauthenticated process template, the Next Step button on the Services tab was missing. This Next Step button would have taken users to the Request Details page. 

Fix: Modified functionality so that the Next Step button appears and users can advance to the Request Details page.
