EMS Desktop Client Release Notes V44.1 Update 4 (EMS Desktop Client)

This section describes updates to your EMS software. 

See Also: Contacting Customer Support and Updating to the Latest Version of EMS

Name: V44.1 Update 4

Release Date: September 19, 2016

Patch #:  2016.09.15.0

Build #: 44.1.266

 

Enhancement 

Area

Description

Timetables

Timezone Adjustments for Caracas, Magadan, Casablanca, Tomsk

Caracas, Magadan, and Casablanca have moved timezones. The timezone for Tomsk was missing.

Fix: Updated timetables in EMS to ensure that the changed and new timezones are reflected in all areas of EMS.

23265

Fixes 

Area

Description

Email Confirmations

Email Confirmations Missing Digits

When sending email confirmations with systems that had the "Display Messages as HTML" option activated, digits to the right of decimal were missing from the emails.

Fix: Modified email triggers so that digits are no longer omitted from email confirmations in this situation.

23263

Campus > Course Updates

Slow Performance When Viewing Course Updates

When EMS Campus users opened the Dashboard, clicked on Notifications, and then Course Updates, the page loaded slowly.

Fix: Modified functionality to improve performance. 

23264

Upgrade from V43 to V44.1 (Event Coordinators Only)

Slow Performance On Upgrade for Event Coordinators

When upgrading, Event Coordinators experienced slow performance. 

Fix: Resolved issue so that upgrade does not produce slow performance.

23261

Room Utilization Report

Report Showing Holidays as Booked

The report was showing bookings on holiday dates.

Fix: Modified the report to exclude bookings on holiday dates.

23262

Web Deploy Only > Create New Department

 

Create New Department Without Logo Causes Error

When creating new departments without logos, the error "Value cannot be null. Parameter name:s" occurred and prevented users from completing the task.

Fix: Resolved issue so that users can now add departments without logos.

23279

CISCO TMS Service

Retry on CISCO TMS Service Causes Error

When users selected one of the bookings in the Cisco TMS Activity window and then clicked the Retry button, the following error occurred: 

Conversion from string "Default" to type 'Integer' is not valid.

Fix: Modified the screen overlay so that the error no longer occurs and the Retry button works as expected.

Note: This fix is delivered as a new DLL and a new overlay script.
