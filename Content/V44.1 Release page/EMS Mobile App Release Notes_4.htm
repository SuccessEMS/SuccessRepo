EMS Mobile App Release Notes V44.1 Update 4 (EMS Mobile App (IPA))

This section describes updates to your EMS software. 

See Also: Contacting Customer Support and Updating to the Latest Version of EMS

Name: V44.1 Update 4

Release Date: September 19, 2016

Build #: 44.1.430

Mobile API #: 44.1.187.0

 

Area

Description

Login > Create Account

EMS Human Resources Toolkit Users Credentials Not Imported Correctly

When EMS Human Resources Toolkit users attempted to log in to EMS Mobile App, security template settings did not affect permissions as expected. The Create Account button was available to users who should not have access to this feature.

Fix: Modified functionality so that Administrators can control the button's availability using the "Create \Edit An Account" role under Configuration > Everyday User Applications > Everyday User Security Templates in EMS Desktop Client.

23080

User Interface

Create/Edit Reservation: Required Fields Unmarked for Long Field Names

When working with a booking, fields that had long field names were not appropriately marked with red to indicate when they were required.

Fix: Modified display so that required fields show a red bar on the left side regardless of field name length. 

Calendar: "Load more" Button Displays When No More Events Exist

When viewing future events, the button appeared even when no more events existed.

Fix: Changed validations so that the button only appears when there are more events to view.

"Reservation Created" Message Repeats After Locking/Unlocking Devices

During the reservation creation process, if users' devices locked and then unlocked while the user was navigating back to the home page, the message "Reservation Created" appeared multiple times.

Fix: Changed the message to only display once after saving changes.

23171, 23124, 23029

Create Reservation > Change Room

Changing Room Changes Timezone

When users changed the room to a new building on a reservation, the time zone changed and an error message presented.

Fix: Modified behavior so that the error message no longer presents. 

23065

Create Reservation > Add Attendee

Adding Attendee to Reservation When Add to Calendar is Disabled Causes Crash

When users added attendees to reservations that had the option to add the event to the calendar disabled, the application crashed.

Fix: Changed functionality so the crash no longer occurs, and added a descriptive error message to clarify when adding the event to the calendar has been disabled.

23366

Edit Outlook© Booking

Editing Outlook© Booking Causes Error

When users edited bookings from Outlook© that had been made using a booking template enabled for both EMS Mobile App and EMS for Outlook, the following error occurred: "Error Loading Template."

Fix: Modified functionality to ensure that users can edit this type of booking without the error. 

Cannot Save Changes to Outlook© Meeting

When editing a booking that did not allow "Add to my calendar," users were unable to save changes.

Fix: Modified validations to ensure that users can save changes in this situation.

23364, 23367

Search

Clicking Search Button Repeatedly Opens Multiple Search Windows

When users clicked the Search button multiple times, multiple search windows launched.

Fix: Modified behavior so that only one search window opens.

23006

Calendar

Calendar Fails to Refresh After Booking Cancelation

When users canceled a booking the Calendar did not refresh to show the cancelation. 

Fix: Modified the page to auto-refresh after a booking cancelation.

Location Name and Details Frozen While Scrolling

When users scrolled up and down, the location name and other details did not scroll.

Fix: Modified display so that all elements scroll as designed. 

Bookings Not Visible When Toggling Back to Home Page

When users navigated to the calendar in schedule (grid) view and then back to the home page, bookings did not show on the home page.

Fix: Modified the display to ensure that bookings are properly listed on the home page. 

23069, 23028, 23386

Check-In

Outdated Parameters for Check-In and Building Proximity

Parameters that controlled user Check-In from EMS Desktop Client for how building proximity was handled during check-in were outdated:

  1. MOBILE_CheckinProximityDistance - Mobile Check In Proximity Distance
  2. MOBILE_CheckinProximityUnitOfMeasurement - Mobile Check In Proximity Unit of Measurement
  3. MOBILE_ProximityDistance - Mobile building proximity distance
  4. MOBILE_ProximityUnitOfMeasurement - Mobile building proximity unit of measurement

Fix: Modified parameters so now the following exist instead:

  1. MOBILE_CheckinProximityDistance - Mobile Check In Proximity Distance
  2. MOBILE_ProximityUnitOfMeasurement - Mobile proximity unit of measurement

23253

iOS Only: Home Page

Today's Events Not Displaying Automatically

When users on iOS devices launched the application, today's events were not automatically listed. 

Fix: Modified the display to list today's events for the current user as designed. 

23378
