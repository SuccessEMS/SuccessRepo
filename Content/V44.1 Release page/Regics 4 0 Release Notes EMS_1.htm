Registration

Database Blocked When Hosting Over 1,300 Registrants

When users created an event using a large number of User Defined Fields, and over 1,300 registrants accessed the system in less than an hour, they experience errors due to database locks and could not complete registration.

Fix: Added functionality to prevent database locks in this situation and prevent the error.

17408

Update Code to Use the HTTP POST Method 

Regics has been updated to no longer use the less secure HTTP GET methods for transaction requests because these methods do not adhere to current TLS protection requirements. It now uses HTTP POST. Modified functionality so that the error does not occur and events with dependencies can be copied successfully.

Manage Attendees

When users sent an email to a registrant that wasn't the group owner using Participant field in the HTML editor on the Manage Attendees page, the email was incorrectly sent to the group owner.

Modified the Participant field to use the email address the user specifies.

 

21916

Credit Card Payment

When paying for an event by credit card with Authorize.net, the payment showed as declined externally even though internally it showed as a confirmed payment 

Modified the payment to show as confirmed both externally and internally.

 

17124

Partial Paypal Refunds

When users issued partial refunds via PayPal, full refunds were sent in error.

Modified validations to ensure that the refund amount in Regics matches the refund issued via PayPal.
