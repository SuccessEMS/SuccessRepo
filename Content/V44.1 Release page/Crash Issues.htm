Crash Issues

This section categorizes resolutions to issues that cause the EMS Mobile app to crash.

Numerous Screen Clicks Cause Crash 

Reported Issue

Issue Resolution

Clicking to Select Screen Items Many Times

When users clicked many times on screen items such as "+" or the "Book Now" button next to a template, EMS Mobile App could crash with the error "unfortunately, EMS has stopped" or "EMS isn't responding".

Modified functionality so that when users click too many times on a screen item, EMS Mobile App continues to function.

Picking Unassigned Groups During Booking Causes Crash 

Reported Issue

Issue Resolution

New Reservation > Group Lookup

When making a new reservation using a template that allows Group lookup, when users added a group to the reservation that was not assigned to their user profile, EMS Mobile App could crash. 

Modified functionality to ensure that EMS Mobile App does not crash during Group lookup and furthermore, assigns the Group to the user's profile (when enabled). 

 Saving Filters Causes Crash 

Reported Issue

Issue Resolution

Home>Browse Locations>Save Filters

Clicking on the Save Filters button prematurely (while locations are still loading) could produce the error, 

"Unfortunately, EMS has stopped"

Modified functionality to prevent the Save Filters button from becoming available until locations are loaded, thus preventing the error.
