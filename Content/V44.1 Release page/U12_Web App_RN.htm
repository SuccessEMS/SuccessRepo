Skype for Business

User Authentication for Skype for Business Online

When users toggle Skype for Business for the first time, they will be presented with a Skype for Business authentication form. If users successfully authenticate, a valid token will be generated and will not be required to authenticate again when adding Skype for Business to bookings in the EMS Web App. If users fail to authenticate, the Skype for Business toggle will be disabled. 

User Authentication for Skype for Business On Premise

When users toggle Skype for Business for the first time, they will be presented with a Skype for Business authentication form. If users successfully authenticate, a valid token will be saved for 8 hours. If users fail to authenticate, the Skype for Business toggle will be disabled. 

Skype for Business Meeting Details 

Users adding Skype for Business to an Exchange-enabled reservation will be able to view the meeting details in the Reservation Details.

Users Can Add Skype for Business to Reservation Using Toggle

Users can add Skype for Business to their reservation by using the Skype Meeting toggle on the first and last tabs of the Room Request screen.  

Skype Toggle Appears if Skype for Business Has Been Enabled (Desktop View)

Skype for Business toggle appears if the Enabled for Skype parameter is set to Yes and the template is enabled for Skype (in Desktop View).

Skype Toggle Appears if Skype for Business Has Been Enabled (Mobile View)

Skype for Business toggle appears if the Enabled for Skype parameter is set to Yes and the template is enabled for Skype (in Mobile View).

All Skype for Business Details—Including Call-in Details—Will Be Included in Email Notifications to Attendees

If Skype for Business is enabled, email notifications for reservations including Skype for Business meetings will include the call-in and meeting details.

Users Receive Notification When Skype for Business Is Added or Removed from Reservation

During the creation of a reservation, users will receive a notification that Skype information will be added or removed from their reservation when they toggle Skype for Business off/on.

 Fixes 

Area

Description

Manage Services

Services for Reservations Created in Request a Room Template Could Not Be Edited through Manage Services

When users created a reservation using the Request a Room template and added Services, those Services could not be edited in Manage Services. No Save Changes button appeared once changes were made. 

Fix: Modified functionality so that services can be edited in Manage Services. 


Contacts

Multiple Contact Instances from Web App

Manually created or modified Outlook contacts were sometimes appearing multiple times in the EMS Web App. Invitation will not send out to all attendees.

Fix: Modified functionality so that only a single instance of the contact created in Outlook will appear in EMS Web App. 

Floor Plan

Icons Not Scaled to Floor Plan Diagram

When users viewed the Floor Plan Configuration, the icons were not scaling to the diagram. Icons were only appearing normal when zoomed in. 

Fix: Modified functionality so that icons appeared the appropriate size.

Floor Map Not Appearing in Chrome or FireFox Browsers

Users attempting to search for rooms in the Floor Map tab could not see floor plan. 

Fix: Modified functionality so that floor plans immediately appear. 

Create a Reservation

Users Could Not Book Buildings Due to Unrelated Building Closure/Holiday

Users trying to book a building were receiving an error stating, "Request violates building hours," despite the fact that the building closure or holiday was not related to the selected building. 

Fix: Modified functionality so that buildings can be booked on dates when there is no building closure or holiday set, despite a different building having a closure or a holiday on that day.

Room Availability

Incorrect Room Availability Data for Recurring Meetings 

When attempting to book recurring meetings, the data under room availability was not correctly reflecting actual room availability. 

Fix: Modified functionality so that the actual availability of rooms is reflected in the room availability column when booking a recurring meeting. 

Create a Reservation

Users Were Allowed to Create Bookings in Closed Buildings

Users creating a recurring reservation were allowed to book buildings that were closed during the recurrence date range. 

Fix: Modified functionality so that customers can restrict whether the EMS Web App allows users to book events in closed buildings. 

Services

After Deleting a Room From a Reservation, Room Still Listed on Services Page

When users deleted rooms from a reservation created in the Web App, those deleted rooms and services were still appearing on the Services Page.

Fix: Modified functionality so that only services and resources for active rooms in the reservation appear.

Browse Events
 

Users Received an Error When Clicking on Room Name in Daily List

Users attempting to view room details by clicking on the room name under Browse Events Daily List were instead receiving an error.

Fix: Modified functionality so that users can view room details when clicking on the room name under Browse Events Daily List.

Custom Link
 

Confirmed Bookings Marked as Canceled in Custom Links

When viewing bookings through custom links, the word Canceled was appearing before the name of confirmed events. The events were not canceled and these same events were showing normally in Browse Events. 

Fix: Modified functionality to ensure the bookings do not appear as Canceled when viewed through the custom links.

Create/Edit Reservation
 

Apostrophes Were Not Displaying Correctly in First Contact Name

In the 1st Contact Name field, apostrophes were not appearing correctly. 

Fix: Modified functionality to ensure apostrophes were appearing correctly. 

Edit
 

Save Changes & Cancel Buttons Not Working When Editing an Existing Service Order Time or Estimated Count

When attempting to edit a service order or estimated count, the Save Changes and Cancel buttons were not appearing to commit the change. 

Fix: Modified functionality so that the Save Changes and Cancel buttons appear underneath the service order details.

Services
 

Unauthenticated Everyday User Request Cannot Request Resources Part of Groupings

Unauthenticated everyday users were unable to request a resource that was part of a grouping. The Grouping would not expand and the user was unable to view or add resources for the Grouping.

Fix: Modified functionality so that the Grouping expands and displays the resources for the reservation.
